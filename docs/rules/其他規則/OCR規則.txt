【OCR 流程規則】

## 一、基本原則

1. 所有圖片、掃描 PDF、image-based PDF、拍照圖面、材料表截圖、手寫單、工程圖等附件，只要需要讀取文字或表格，皆視為 OCR 任務。
2. 若文字或圖面有旋轉，須先在判讀時旋正後閱讀。
3. 中文一律保留繁體中文，不得轉為簡體。
4. 不得從檔名推論尺寸、規格、數量、價格或其他資料。
5. 若中文字、數字或符號視覺上不明確，須列出候選值或標示「未確認」，不得自行猜測。

## 二、OCR 工具使用規則

1. 需要讀取 PDF、圖片或圖面文字 / 表格時，AI 必須先呼叫 `run_file_ocr`。
2. `run_file_ocr` 由 backend 透過 PaddleOCR MCP 執行；AI 不得改用內建 OCR。
3. PDF 必須以整份文件送入 `run_file_ocr`，不得只擷取部分頁面。
4. 圖片也須先用 `run_file_ocr` 取得文字 / 表格結果。
5. 只有加工內容：孔數、折邊、割型、開槽連續邊長、切角、缺口、輪廓等非文字幾何判斷，才可由 AI 視覺判斷，且須標示信心程度。
6. 若無法呼叫 `run_file_ocr` 或 OCR 失敗，須直接說明失敗來源、錯誤摘要與需人工補資料的位置，並改用內建的 OCR 模組處理。

## 三、輸出格式

AI 回覆須使用繁體中文，並以 Markdown 表格整理：

1. 可報價資料
2. 低信心項目
3. 來源標籤
4. 需人工複核欄位

若來源表格可辨識為 N 列，輸出表格應盡量對應 N 列；即使部分欄位不清楚，也必須保留該列，缺失欄位填「未確認」或標示低信心。

## 四、可報價資料欄位

整理 OCR 結果時，表格應盡量包含下列欄位；來源沒有或無法確認可留空白：

| 欄位           | 說明           |
| ------------ | ------------ |
| 項次           | 原表格項次或自行排序標示 |
| 品名 / 材料名稱    | 板材、型鋼、零件名稱等  |
| 規格           | 原始規格文字       |
| 尺寸           | 長、寬、高等尺寸     |
| 厚度           | 如 6t、9t、10t  |
| 數量           | 件數、支數、組數等    |
| 單位           | PCS、支、片、組等   |
| 加工內容         | 切割、鑽孔、折彎、開槽連續邊長等 |
| 孔數           | 若需由圖面判斷，標示信心 |
| 折邊           | 有 / 無 / 未確認  |
| 切角 / 缺口 / 開槽 | 幾何加工資訊       |
| 備註           | 原圖備註或判讀說明    |
| 來源標籤         | 來源頁面、圖片或表格位置 |
| 信心程度         | 高 / 中 / 低    |
| 是否需人工複核      | 是 / 否，並簡述原因  |

## 五、常見 OCR 誤判修正

1. 鋼材規格尾端的 `t` 通常代表厚度，PaddleOCR 可能誤判為 `1`。
2. 若 OCR 顯示如 `341×500×101`，但同欄位或相鄰列皆為板厚格式，應判斷可能為 `341×500×10t`，並標示低信心或人工複核。
3. 若 OCR 顯示乘法公式左右不一致，應以 operands 重新計算，不採用錯誤等號右側結果。
   * `1×12=6` 應修正為 `1×12=12`
   * `1×2=4` 應修正為 `1×2=2`
4. 若來源數字本身仍不確定，須保留低信心，不得改成確定值。

## 六、需人工複核情況

下列情況必須標示「需人工複核」：

1. 文字模糊、缺字、被遮擋、反光、陰影或解析度不足。
2. `t`、`1`、`l`、`I` 等字元無法確認。
3. 規格、厚度、數量、孔數或尺寸不完整。
4. 公式重新計算後，來源數字仍可能有誤。
5. 表格列數無法與原圖完全對應。
6. 孔數、折邊、切角、缺口、開槽、輪廓等需由 AI 視覺判斷。
7. 使用者後續更正 OCR 表格資料時，須以最新使用者訊息為準重新整理，不再呼叫 `run_file_ocr`。
