【OCR 規則】

一、主 Agent 流程

1. 需要讀取圖片、掃描／影像 PDF、圖面、截圖或手寫附件的文字／表格時，視為 OCR；完整處理來源，不漏頁。
2. 優先使用本輪同 file key 已提供的 organized OCR Markdown／PaddleOCR 結果。只有資料缺失、失敗，或使用者明確要求重做 OCR 時，才呼叫 `paddleocr_vl`；不得因後續確認、報價或重整表格自行重做。
3. PaddleOCR 失敗或無可用文字／表格時，才使用 AI OCR／vision fallback，並標示「AI OCR fallback」、失敗摘要與不確定欄位。Fallback 不算 PaddleOCR 成功；資料仍缺失或使用者明確要求重做時可再嘗試 PaddleOCR。

[ocr_shared]
1. 旋轉的文字或圖面先旋正再判讀；中文一律保留或轉繁體中文。保留來源列、原始規格與來源頁碼。
2. 文字、數字、符號、尺寸、厚度、數量、重量不清時，不猜測，禁止用「約、略、大約、約略」等近似詞代替；欄位留空，並在`備註`列候選或說明低信心／人工複核。
3. `t` 被辨成 `1` 等明顯 OCR 誤判時直接修正，並在`備註`記錄原辨識值、修正值與原因。例如依欄位脈絡將 `341×500×101` 修正為 `341×500×10t`。
4. 公式結果與 operands 不一致時，直接以 operands 重算修正，並在`備註`記錄原公式／結果與修正值；operand 本身不清楚時欄位留空並寫備註。
5. 使用文字標註與可見幾何判讀孔數、折邊、割型、開槽（連續邊長）、切角、缺口與輪廓；逐項保留來源、計量方式與信心，不確定時欄位留空並寫備註，不得省略或用近似詞。
6. 孔：記錄每件孔數、件數、總孔數、孔徑／孔型；`總孔數 = 每件孔數 × 件數`。依尺寸標註、孔符號與重複排列計數，不把中心線、尺寸端點或文字符號當孔。
7. 折邊：每一道明確折線／折彎標註計一次，記錄角度、折邊長度與重複數；無法分辨折線與輪廓線時標複核。
8. 割型、切角、缺口、輪廓：依圖面實際加工線與數量分列，記錄切割方式、線段／圓弧、角度及重複特徵；不得用外接矩形代替實際輪廓。
9. 開槽連續邊長：沿實際開槽加工路徑加總直線與圓弧長度；封閉槽計完整周長，開放槽只計明示加工邊。相同槽乘槽數與件數，不重複計共用邊；尺寸不足時欄位留空並寫備註。
[/ocr_shared]

[ocr_organizer]
1. 只依提供的 raw OCR 與 file／chunk metadata 整理一張 chunk-local Markdown 資料表，不補查、不報價、不輸出 final file-key 標題。
2. 每筆來源列獨立保留，不合併同品名／規格；每列至少標記來源頁數、項次、件號、圖號或其他可追溯代號。缺值留空，無法判斷時於`備註`說明，不用近似詞。
3. 報價相關資料視來源保留：項次、品名／規格、尺寸／厚度、數量／單位、孔數、折邊、割型、開槽連續邊長、切角、缺口、輪廓、來源頁碼／代號、信心與複核原因。
[/ocr_organizer]

二、最終 OCR Markdown

1. 最終回覆依 file key 分組；同一 file key 的所有 chunk 合併成一張 OCR 結果確認表，但每筆來源列保持獨立，不得彙總或合併同品名／規格。不同 file key 不混表。
2. 標題固定為 `## OCR 結果確認表：<filename>（file:<id>）`。
3. 保留每一來源列；每列至少包含來源頁數、項次、件號、圖號或其他可追溯代號。欄位依來源與報價需要包含品名／規格、尺寸／厚度、數量／單位、孔數、折邊、割型、開槽連續邊長、切角、缺口、輪廓、信心與複核原因；缺值一律留空。
4. `t／1`、公式結果等已修正內容直接使用修正值，並在同列`備註`保留原辨識值、修正內容與原因。
5. 模糊、缺字、遮擋、反光、低解析、字元歧義、規格／數量／尺寸不完整、列數不一致或需 vision 幾何判讀時，在同一表的信心／複核欄標示；不另要求第二張複核表。
