【OCR 主 Agent 整理與合併規則】

[ocr_main_merge]
1. 依 file key 分組，同一 file key 的所有 chunk Markdown 合併成一張 OCR 結果確認表；不同 file key 不混表。
2. 合併時保留每筆來源列，不彙總、不合併同品名／規格，並維持來源頁數、項次、件號、圖號或其他可追溯代號。
3. 欄位以各 chunk 已整理內容為準；缺值保持空白，信心與複核原因保留在同一列，不重新執行 PaddleOCR、不查價、不產生正式報價。
[/ocr_main_merge]

[final_ocr_markdown]
1. 標題固定為 `## OCR 結果確認表：<filename>（file:<id>）`。
2. 每列依來源與報價需要包含品名／規格、尺寸／厚度、數量／單位、孔數、折邊、割型、開槽連續邊長、切角、缺口、輪廓、信心與複核原因；缺值一律留空。
3. OCR 或 Vision 已修正的內容直接使用修正值，並在同列`備註`保留原辨識值、修正內容與原因。
4. 模糊、缺字、遮擋、反光、低解析、字元歧義、規格／數量／尺寸不完整、列數不一致或需 Vision 幾何判讀時，在同一表的信心／複核欄標示，不另建立第二張複核表。
[/final_ocr_markdown]
