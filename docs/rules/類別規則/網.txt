網類 parser、query 與計價規則

1. Query
- 一般首查只使用 `category: "網"` 與可確認的 `subcategory`、`material`，不使用 `unit` filter。即使訂單指定計價單位，unit也只用於取得候選後的計價判斷，不送入query。
- 例如 `{"category":"網","subcategory":"菱形網","material":"錏"}`、`{"category":"網","subcategory":"刺網","material":"白鐵"}`。
- 網類資料量小，不使用 `keyword`，也不新增尺寸 query params，不帶 `unit`、`stockLengthMm` 或訂單長度 filter。未指定 `limit` 時維持通用預設30；將返回的 `category/subcategory/material` 候選交給AI判斷。
- AI依候選的 structured fields 與完整 `product_name` 判斷線徑、孔目、孔型、整片／整捲尺寸、可拉長度與是否足夠裁切；預設不切清。
- `kg`、`尺`、`inch`、`mm`、`ST304` 等單位、鋼種或代號不得轉成 `keyword`。

2. Parser正規化
- 尺、inch、mm原文保留在品名／備註；供候選比較的mm值四捨五入為整數。
- `ST網2尺x100尺33#(0.2)16目(1.6)`：2尺是網寬606mm，100尺是網長30300mm，16目是每英吋網孔數，0.2是線徑mm，1.6是孔徑mm。
- `ST網4尺x100尺(16目)`：4尺是網寬1212mm，100尺是網長30300mm。單獨標示2尺通常是網寬，寫入 `sheetWidthMm=606`，不是 `lengthMm`。
- `點焊鋼絲網5.5 15x15 2Mx3M(6)` 或 `6.0足 15x15 2Mx3M(6)`：5.5／6.0足是線徑mm，15x15是cm網孔並正規化為150x150mm，2Mx3M是每片2000x3000mm，尾端(6)是每捆／束6片包裝數，不是kg或單重。來源欄位若把6誤寫為單重，parser必須清除。
- `錏浪型網 8#(3.6)x38mm □孔`：8#是線號，3.6是線徑mm，38mm是38x38正方孔；不得採用污染的inch衍生值。品名包含浪型網時，subcategory一律修正為浪型網。
- `黑鐵刺網105M(8.2KG)`：105M是整捲長度105000mm，8.2KG才是整捲重量。
- `鐵網的固定片(ST304)`：ST304是材質代號，不得解析為厚度304mm。

3. Unit與計價
- 訂單中的㎡、m2、才、捲、卷、丸、張、片、只保留供取得候選後的計價判斷，不作價格query filter。
- 全部預設不切清，只採direct tier。AI從全部候選中選擇足夠裁切且成本合適的材料。
- `m2/㎡` 是唯一可按實際小數數量計價的單位：需求面積㎡直接乘每㎡單價。
- `才`：以1才=0.091809㎡換算，無條件進位成整才後乘direct單價。
- `丸`：依候選整捲可用長度計算捲數，不足一捲仍按一整丸，不按使用長度比例拆價。
- `片`：依候選整片尺寸配料並進位成整片；`只`按整數配件數。
- 網類目前沒有可安全套用的ratio-only公式；沒有direct價時送人工複核。
