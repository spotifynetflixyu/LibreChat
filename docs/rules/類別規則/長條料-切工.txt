【長條料/切工類別規則】

一、適用範圍
適用鐵軌、H型鋼、I型鋼/工字鐵、角鐵、槽鐵、平鐵、圓條、鋼筋、圓管、方管、扁方管等需裁切長條料。

二、材料是否切清
平鐵、圓條、鋼筋、圓管、方管、扁方管由採用 price row 的有效 `unit` 決定材料計價：`Kg` 按實際總 Kg 計價；direct price 的 `支`、`只` 預設不切清，按素材整支數量 × direct 單價計價。`ratio_only` 且來源 `unit: 支` 的候選已由 backend 轉成有效 `unit: Kg` 與一般 `tierPrices`，AI 必須按重量計價，不得再按支計價或判讀 raw `price_ratio`。若訂單成品長度需要裁切，再依刀數另加 candidate-aware 切工費；整支原長交貨為 0 刀，不加切工費。H型鋼以 Kg 為主；其餘長條料維持既有類別規則。
不切清時，材料費依素材長度配料計算，需列素材長度、素材支數、可裁成品數、餘料長度/重量。平鐵、角鐵、圓管、圓條、扁方管、方管、槽鐵以 `product_name` 明確長度優先，名稱沒有長度時固定用6M，不標低信心。這些預設6M材料的 query 不帶 `stockLengthMm`，AI從返回的6000、12000或其他可用素材長度做不切清配料；訂單成品長度只用於裁切計算，不作母材長度 query filter。其他長條料素材長度不明時，用類別規則預設；仍不明才標低信心。鋼筋不補6M；鐵軌依K級別catalog明示長度，不可全類預設6M。

三、切工基本價查法
`search_price_candidates` 將一般價格 queries 與一次 `steel.cutting_prices` category catalog 查詢並行執行；兩邊返回後，backend 依成功材料 candidate 的規格做 code-level 篩選，再把結果放在頂層 `cuttingPrices`。AI 不另建切工 query。

DB 切工查詢仍只使用類別包含條件，不使用 limit；輸出時只保留 `quoteEligible: true` 且規格唯一的成功材料 candidate 對應規格列與適用補充規則。材料 `no_match`、候選不可報價、缺少可用規格或多個候選對到不同切工 key 時，不附該 query 的切工，也不得回退同類全部 catalog。H型鋼與 I型鋼/工字鐵以高度×翼寬 exact match，H 14mm 加工另由完整斷面翼板厚判定；管類優先用 candidate `nominalInch`，沒有 nominal 時只使用 backend 核准的 metric-to-inch 對照；角鐵用邊長；槽鐵通常用高度×寬度，但 `150x9.0` 特例用高度×腹厚；平鐵用實際寬×厚。資料表 inch 仍用 1 inch = 25.4mm 精確換算，不可使用最近尺寸。

自動對應：

| 材料 category | 切工包含搜尋詞 |
| --- | --- |
| H型鋼 | H型鋼 |
| I型鋼/工字鐵 | 工字鐵；只限材料 candidate 可報價時 |
| 圓管、方管、扁方管 | 鐵管 |
| 圓條 | 鐵管；只保留圓條不切/外切限制，不借用基本價 |
| 角鐵 | 角鐵 |
| 槽鐵 | 槽鐵 |
| 平鐵 | 平鐵，可匹配 `鐵板/平鐵` |

`cuttingPrices[].queryIds` 只保留成功且實際命中該切工規格的 backend `q1`、`q2`…，並與重建後的 `sourceCategories` 一起對回原訂單列；`no_match` 或不可報價 query 不得出現在 cutting provenance。鐵軌沒有切工列；鐵板沒有獨立自動切工價，兩者都不得借用相近類別價格。只有訂單明確包含獨立加工品項，且該加工未由 candidate-aware `cuttingPrices` 返回時，才以 `category: "加工/切工"` + 對應 `subcategory` + 加工名稱/規格 `keyword` 查詢。

材料首查規格範例（每類均已用 live `steel.prices` 驗證能返回候選）：

- H型鋼：`{"category":"H型鋼","keyword":"250x250x9/14","stockLengthMm":["6000","9000","10000","12000"]}`
- I型鋼/工字鐵：`{"category":"I型鋼/工字鐵","keyword":"200x100x7/10"}`；現有候選不可報價，返回後人工複核。
- 平鐵：`{"category":"平鐵","material":"黑鐵","keyword":"50x6mm"}`
- 角鐵：`{"category":"角鐵","material":"黑鐵","keyword":"50x6mm"}`
- 槽鐵：`{"category":"槽鐵","material":"黑鐵","keyword":"200x90x8/13.5"}`
- 圓管：`{"category":"圓管","subcategory":"鋼管","material":"黑鐵","keyword":"4in"}`
- 方管：`{"category":"方管","material":"黑鐵","keyword":"100x6"}` 可取得可報價候選；圖面 100×100×6 的價格 keyword 只寫一次邊長。150×150×6 仍查 `150x6`，但目前候選不可報價，須人工複核。
- 扁方管：`{"category":"扁方管","material":"黑鐵","keyword":"40x80x3"}`
- 圓條：`{"category":"圓條","material":"黑鐵","keyword":"10mm"}`
- 鐵軌：`{"category":"鐵軌","keyword":"12K"}`
- 鋼筋：`{"category":"鋼筋","keyword":"9mm"}`；現有rows全部no_price，返回no_match後人工複核。

以上範例只使用通用產品名稱與規格。專有品名、廠牌、系列名或來源 DB token 只有用戶明確指定時才可原樣加入 keyword，不得由候選 product_name 反推。

管類通用 query 規則：

- 圓管、方管、扁方管的 `category`/`subcategory` 已表達產品類別，keyword 不得再加入圓管、鋼管、方管或扁方管等類別詞，只放 category/subcategory 尚未表達的斷面與厚度規格。
- 公制尺寸統一使用 `mm` 語意；英制 nominal 保留原分數或 inch 規格。不得把用戶的英制分數只改成近似公制後查詢，也不得把兩個不一定同時出現在來源資料的等價尺寸強制放在同一個 AND keyword。
- 尺、inch、mm或其他長度單位的原始表示保留在品名／備註；正規化為mm供query或候選比對時四捨五入成整數，不以小數位匹配。台制尺採1尺=303mm。
- 素材長度不放 keyword。只有用戶明確要求特定整支長度，而且候選資料具有結構化長度時，才使用 `stockLengthMm`；不得為了長度排除缺少結構化長度但規格、材質與 unit 可用的候選。

圓管 query 規則：

- 圓管未指定材質時預設 `material: "黑鐵"`；只有用戶明確指定其他材質時才改用對應 material。
- keyword 只放來源規格的直徑×厚度；公制統一分隔符並保留直徑 `mm`，英制分數保留原分數/inch。來源只給直徑時只放直徑，不猜厚度，也不得同時加入等價的英制與公制尺寸。
- 括號數字是單位重，只存單重/備註，不得加入 keyword；素材或成品長度也不得加入 keyword。
- 黑A、黑B、白A、美亞等來源／專有型態只有用戶明確指定時才可原樣加入 keyword，不得由候選 product_name 反推。
- `連料`、`太陽片` 只有用戶明確指定時才查；一般圓管 query 不得帶入。連料尺寸不得解析成管徑×厚度，太陽片的溝數／代號也不得污染一般圓管尺寸。

圓鐵（價格 `category: "圓條"`）query 規則：

- 未指定材質時預設 `material: "黑鐵"`；只有用戶明確指定白鐵/ST/不鏽鋼、錏/鍍鋅/熱浸鍍鋅等材質時才改用對應 material。
- keyword 只放直徑：公制裸數字、`Ø`、`φ`、`m/m` 統一成 `{直徑}mm`，並移除無意義的小數尾零；英制分數正規化斜線與引號後保留原分數，例如 `3／8吋` 使用 `3/8`，不得只換成 `9.525mm`。
- 不得在 keyword 重複加入圓鐵或圓條，也不得加入長度、整支、切、過磅、Kg、支或重量備註。長度只用於配料；明確整支長度可依前述條件使用 `stockLengthMm`。
- 用戶未指定產品型態時，同直徑候選採用順序為：一般圓鐵/圓條（包含不含中碳字樣的磨光圓鐵/磨光圓條）→ 中碳 → 磨光中碳。前一順位沒有可報價且 unit 合用的候選時才用下一順位。
- 用戶明確指定磨光、中碳或磨光中碳時，以指定型態為準，該通用型態詞才可與直徑一起放入 keyword；`中碳` 與 `磨光中碳` 必須分開判定，不得因文字包含關係把明確中碳需求自動改採磨光中碳。
- 例：一般黑鐵 12mm 使用 `{"category":"圓條","material":"黑鐵","keyword":"12mm"}`；黑鐵 3/8 使用 `{"category":"圓條","material":"黑鐵","keyword":"3/8"}`；明確磨光中碳 12mm 才使用 `{"category":"圓條","material":"黑鐵","keyword":"磨光中碳 12mm"}`。

四、基本公式
切工費 = (基本切工 direct 單價 + 加價) × 刀數 × 數量基準。
若價格列已含加價或為式價，依 price row/category rule 單位優先，不得重複加價。價格空白、0、未確認或待確認者不可填 0，列人工複核。

五、刀數計算
一個切口 = 1 刀。

未修頭尾時：

- 成品長度 = 素材長度，且不需裁切：0 刀
- 一支素材切成 n 支且無餘料：n − 1 刀
- 一支素材切成 n 支且有餘料：n 刀
- 不易逐支判斷時，可用成品支數暫估刀數，但需標註「切刀數以成品支數估算」

切頭/切尾：

- 切頭：每支素材 +1 刀
- 切尾：每支素材 +1 刀
- 切頭尾：每支素材 +2 刀
- 若明確說每支成品修頭尾，則每支成品 +2 刀
- 對半切不修頭尾 = 1 刀
- 對半切 + 修頭尾 = 3 刀

六、額外加價/限制規則

H型鋼/I型鋼/工字鐵：

- H型鋼斜切：切平行斜刀 = 基本價 ×2 −10；切梯形斜刀 = 基本價 ×2；翼板切斜 = 基本價 ×2 +50
- H 14mm 以上另計：開槽 KZZB10 A/C/F=140、B=150；沖孔 KZZB11 A/C/F=16、B=17；倒角 KZZB12 A/C/F=140、B=150。KZZB11 是共用沖孔價格，孔徑不作價格 key，不得按 Ø24、Ø22 拆查。
- H型鋼套用工字鐵切工價格時，價格再加50
- 大料、加厚、特殊角度、翼板切斜註記不清時列人工複核

管類通用規則（圓管、方管、扁方管）：

- 黑鐵管類切斜 = 基本價 ×2 −5
- 管類厚度厚另計
- 方管厚度 1.2 以下不切
- 白A、錏方管 +5
- 白鐵 100 以下 +5；白鐵 100 以上 +10
- 1吋以下小方管量少加價

方管 query 規則：

- 未指定材質時預設黑鐵；一般材料 keyword 只放單一邊長x壁厚，例如 `25x1.2`、`3/4x0.8`，不得加入方管類別詞、材質、BA或素材長度
- `正40` 正規化為40，`11/4` 正規化為 `1 1/4`；明示 M/L 長度優先，未標長度固定6M，但長度不得加入材料 keyword
- 太陽片、雨棚架、沖孔窗、連料、U型屬特殊成品；keyword 保留通用成品種類、名義邊長與必要成品尺寸。尺／寸、雙內80、U型110x480不得解析成壁厚或素材長度
- 連料使用 `subcategory: "連料"`；型號或來源專有 token 只有用戶明確指定時才可保留

扁方管 query 規則：

- 未指定材質時預設黑鐵；keyword 只放寬x高x壁厚，例如 `100x200x4`、`15x30x1.5`，不得加入扁方管類別詞、材質或素材長度
- 兩個不同邊長都必須保留，不得比照方管只寫一個邊長，也不得排序後遺失原始寬高配對
- product_name 明示 M/L 長度時優先，未標長度固定6M；`10x20x6M` 的6M只代表素材長度，厚度未知，不得誤填 t6mm

圓鐵專屬規則（價格 `category: "圓條"`）：

- 圓條預設不切；1吋以下圓條不切或需外切，列人工複核
- 圓條只借用鐵管切工 catalog 判斷「圓條不切／外切」限制，不得借用圓管、方管或扁方管的基本切工價

角鐵：

- 白鐵角鐵在黑角鐵基本切工上 +5
- 材質或尺寸不清楚，標低信心
- 未指定材質時預設黑鐵；熱浸鍍／熱進鍍鋅統一為錏／鍍鋅。等邊keyword只放邊長x厚度，不等邊使用subcategory不等邊並保留長邊x短邊x厚度
- BA、2B只用material表達；成型角鐵保留通用特徵成型，素材長度與括號整支重量不加入keyword
- 明示M/L/尺長度優先，一般材料未標固定6M；query不帶stockLengthMm。烤漆12尺保留原文並正規化為3636mm；塑膠腳套等配件不補6M

槽鐵：

- 特長、修頭尾外 +30
- 特短外 +10，暫以 500mm 以內判斷
- 特長/特短界線不明時列人工複核
- 未指定材質時預設黑鐵；熱浸鍍統一為錏／鍍鋅。keyword只放高度x翼寬x腹厚/翼厚，素材長度與括號整支重量不得加入
- 明示M/L長度優先，未標長度固定6M；預設6M材料的query不帶 `stockLengthMm`

平鐵：

- 平鐵未指定材質時預設 `material: "黑鐵"`；keyword 只放寬x厚
- 括號數字是單位重，只存 `unitWeightValue`/備註，不得加入 keyword；素材或成品長度也不得加入 keyword
- 來源或訂單有明確素材長度時優先使用；product_name 沒有素材長度時按6M。direct `支`/`只` 按整件價；`ratio_only + 來源 unit: 支` 使用 backend 轉換後的有效 `unit: Kg` / `tierPrices`，按重量計價
- 白鐵平鐵另計
- 1吋以下平鐵量少加價
- 寬度不明時不可自行補值；厚度未指定時依通用規則，在其他條件相符且可報價的候選中採最小厚度。若無法唯一確認候選，列人工複核。

七、低信心條件
以下不可填 0，需標低信心或人工複核：價格空白、價格為 0、手寫不清、材質不明、未指定厚度且無法依通用最小厚度規則唯一選價、規格無完全匹配、素材長度不明、切清/不切清不明、修頭尾是素材修或成品修不明、特殊角度或斜切方式不明。
