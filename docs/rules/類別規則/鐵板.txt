【鐵板規則】

柱底板、連接板、加勁板、補強板、端板、底板、蓋板等，皆視為鐵板加工成品名稱，不另當鋼材類別。只要有長、寬、厚、數量，即歸入鐵板候選並計重。

鐵板 query 未提供 `unit` 時由 tool 預設 `Kg`；`Kg`、`kg` 正規化為 `Kg`，其他輸入一律轉成 `片`，不拋 validation error。未提供或提供不合規 `material` 時由 tool 預設 `黑鐵`。AI 可提供黑鐵、白鐵/ST/st/不鏽鋼、2B、NO1、HL/沙面、BA/亮面、鋁、錏、鎢、鋅、塑膠；material alias 由 tool 正規化，不合規值不得觸發重查。

只指定白鐵、ST 或 st，未明確表面材質時，厚度小於 3t 由 tool 正規化為 `2B`，厚度 3t 以上（含）正規化為 `NO1`。明確的 HL/沙面、BA/亮面、2B、NO1 直接使用對應 material。訂單未指定厚度時，在其他條件相符且可報價的候選中採最小厚度。

記號判讀：記號 `PL` 後數字為厚度 mm，`*` 後數字為寬度 mm。例：`PL6*80` = 6mm 厚、80mm 寬；材料首查只將厚度放入 `thicknessMm`，成品寬度保留給重量計算。

板材查價優先順序：

1. 使用 `category: "鐵板"`，次類別 enum 為 `網板`、`花板`、`圍籬板`、`檔泥板`、`特殊`；未確認次類別時省略。
2. 首次材料查價使用 `category`、tool 正規化後的 `unit`、`material`、`thicknessMm` 與可確認的 `subcategory`。材質、表面、厚度與板種已有 structured filter 時不得重複加入 `keyword`；`ST50`、`SN400B` 是表單代號，只保留在判讀與備註。
3. 未指定加工關鍵字時，同材質、同厚度、同 unit 的候選依「雷射切割 → 四方切 → 版型切型」順序採第一個可報價類型；tool results 已依此順序排列。使用者明確指定其中一種時只採指定類型，指定類型缺價就送人工複核，不自動 fallback。
4. 依通用 backend 價格輸出契約，`value_state: ratio_only` 且來源 `unit: Kg` 或 `unit: 支` 的任何類別候選，都會輸出有效 `unit: Kg` 並將 `price_ratio` 轉成一般 `tier_price` / `tierPrices` 格式；AI 只讀正常價格欄位，不判斷 ratio key。
5. `unit: 片` 時，厚度相符的所有整片尺寸候選都保留給 AI 比較配料成本。AI 不得把 4x8、4x10、5x10、1M8 等板寬／板長加入 query，也不得為尺寸拆分或重查；成品尺寸只用於比較各候選如何配料最划算。
6. 材質預設造成的不確定，用黑鐵 Kg 單價暫估。

材料與加工要放在同一次 grouped query 的獨立 query，例如：
`{"queries":[{"category":"鐵板","material":"黑鐵","thicknessMm":["15"]},{"category":"加工/孔","subcategory":"鐵板","keyword":"鑽孔","thicknessMm":["15"]},{"category":"加工/折工","subcategory":"鐵板","keyword":"板折 15mm"}]}`
依 `queryResults` 相同順序對回材料、孔與折工。首次材料 query 沒有可用的精確候選時，依最後載入的通用查價方式重試。鐵板沒有獨立自動切工價；「OT板雷射切割／四方切」是材料候選的 Kg 單價，不得再從平鐵切工資料重複加價。只有訂單明列獨立切割加工且材料價未含該加工時，才另送對應加工 query。

重量公式：
鐵板四方理論重 Kg = 長 mm × 寬 mm × 厚 mm × 密度 ÷ 1,000,000 × 數量。
割形、切角、沖孔、折板無展開尺寸時，先用外包四方尺寸；有展開尺寸時優先用展開尺寸。

計價：
板材材料費原則 = 四方理論重 Kg × Kg 單價。找不到可用 Kg 價時送人工複核，不自動改用 M、片或其他單位。
切/切清 = 依成品尺寸理論重 Kg 計價。
不切清 = 只有訂單明確指定「不切清」時，才依整片/素材 piece 或其他整片 direct 單價計價，不用算餘料。
重量與單價保留原始精度，每一材料列的最終小計以 TWD 整數無條件進位。

加工：
切工、孔加工、開槽、折工、其他加工須分開計算。加工單價不明不可填 0，除非 true zero rule 明確成立。系統訂單中材料與加工分列。
