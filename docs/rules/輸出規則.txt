【Steel 報價輸出規則】

一、輸出範圍

1. 只輸出與本輪回答相關的表；每張表須呈現目前可確認的完整結果，不只列出本輪差異。
2. 客戶等級影響價格時，`system_order` 與 `customer_quote` 使用同一等級／計價基準重算。等級不明時依 B→A→C→F→D→E 取第一個有價等級；六個等級皆無價但有 `unit_price_base` 時，以價格B（`計價基準`=2）套用 base fallback。未直接使用價格B時須列入 `manual_review`。

二、通用格式

1. 報價階段第一張表必須是 `system_order` 逐項明細；其他表與說明置後。
2. 無完全匹配時可用相近品項暫估並標低信心；否則保留品項，`system_order`.`單價`留空、`備註`寫「未確認」，`customer_quote`.`小計`寫「未確認」。未知或缺漏價格不得填0；只有使用者明示免計費或已有明確適用的零價規則時才填0。
3. 已知客戶等級且該等級有價時直接採用；該等級缺價時，依 B→A→C→F→D→E 取第一個已有的等級價格 fallback；六個等級皆無價時才用 `unit_price_base`。只要 base 或 A至F 至少有一個價格，該品項即可暫時報價；fallback 不得覆寫原始價格資料。
4. 實際採用任何 fallback 時，須列入 `manual_review`，並在報價表後的補充明寫「fallback 價格，須人工確認」及採用的價格基準。
5. 最終回覆只呈現報價表、必要補充與說明，不附內部處理資料。
6. 每張 Markdown 表標題必須使用 `## <markdown_name>` 並保留原名：`system_order`、`customer_data`、`customer_quote`、`manual_review`；有來源檔案時才輸出 `## <markdown_name>｜<filename>（<file_key>）`，多檔依來源順序以`、`連接，不得以檔名或 file key 取代 Markdown 原名；沒有來源檔案時只輸出 `## <markdown_name>`，不得虛構檔案資訊。

三、`system_order`

1. 欄位固定依序為：`型號`、`品名規格`、`材質編號`、`單位`、`數量`、`單重`、`總數`、`單價`、`計價基準`、`公式編號`、`厚度`、`寬度`、`長度`、`肚`、`類別`、`備註`；不得增欄。
2. `型號`、`品名規格`、`材質編號`、`類別`、`公式編號`精確複製已採用的價格資料。未完全匹配時，無法確認欄位填「未確認」，`品名規格`保留原訂單文字／OCR編號，差異寫入`備註`。`計價基準`只填1至6（A至F），不確定填2。
3. 每個訂單／OCR產品、材料、件號與來源列各自保留材料列；來源項次、OCR編號與輔助資訊寫入`備註`。
4. `數量`、`單重`、`總數`、`單價`、`厚度`、`寬度`、`長度`、`肚`只填純數字；無資料留空，說明寫入`備註`。
5. `肚`只在`公式編號`為DA、DB、DC時填值，並於`備註`標示計算邏輯待確認；其他列留空。
6. 孔、切工、折工、刀、開槽、切角、缺口與其他／外包加工各自列在材料列之外。加工列採用已選定的加工價格資料，`數量`與`總數`填實際計價量；未確認仍保留列並留空未知數字。孔加工列數量合計必須等於已確認總孔數。

四、其他表

1. `customer_data`只在本輪確認客戶名稱、聯絡資訊、價格等級或報價條件時輸出。
2. `customer_quote`固定欄位為`項目`、`說明`、`小計`。材料依來源逐項列出；加工可按孔／刀／切工／開槽等類型合併，但須列來源與計價量，不得併成單一材料費或加工費。不得顯示內部等級、成本、毛利、計價基準或tier；最後一列為`總計`，未確認費用不得混入已確認總額。
3. `manual_review`固定欄位為`來源表格`、`來源件號 / 項次`、`問題欄位`、`目前判斷`、`需確認內容`、`影響範圍`；只列待確認／低信心項目，且不得取代其他表的必要列。
