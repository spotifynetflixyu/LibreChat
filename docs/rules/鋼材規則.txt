lookup_quote_rules 回傳規則

lookup_quote_rules 是鋼材品類、訂單格式、計價方式、配料、true zero、加工與客戶/品類規則的 merged rule lookup。

當 AI 已推得 catalog family、品類、加工類型或配料/加工規則需求時，必須用 lookup_quote_rules 取得規則提示與候選；不要另呼叫獨立公式 lookup tool。

lookup_quote_rules 應回傳 rules array。每個 rule object 可代表特定客戶、特定品類或特定品名/品類推論後的規則，內容至少要讓 AI 判斷：

材料或品類適用範圍。
報價計價單位與計算方式。
配料、素材長度、餘料、是否可切清。
切工、孔加工、開槽、折工與其他加工是否計費。
true zero rule 是否成立。
需要人工確認的缺口與信心降低原因。

公式編號與系統訂單公式填值不由 lookup_quote_rules 提供；它們由 docs/reference/workbook規則.txt 的固定 workbook 規則與 backend semantic data 決定。

backend 不另跑公式計算邏輯，也不以獨立 formula lookup tool 作為 runtime 必要步驟。

C 型鋼專用規則

當 catalog family 或 quote rule 判斷為：

C 型鋼
鍍鋅 C 型鋼
錏 C 型鋼
亞 C 型鋼
c_type

必須套用 C 型鋼專用規則。

規則：

C 型鋼屬於鋼捲抽料 / 成型下料，不視為一般 6M 長條素材裁切料。
不做 6M 長條料配料。
不算素材支數。
不算餘料長度。
不算餘料重量。
材料數量依客戶指定成品長度與支數計算。
若 price row 為 kg 價：用成品總長 × 單位重量 × kg 單價。
若 price row 為 M 價：用成品總長 × M 單價。
若 price row 為支價、式價或其他單位：依價格列單位計算，不可自行轉換。
2C 表示兩支組合，重量與數量需乘 2。
C 型鋼下料長度不另算一般切工。
C 型鋼孔洞預設不另算孔加工費；孔數可記錄在備註。
只有使用者明確說「孔另計」、「切工另計」、「修頭尾」、「特殊切角」，或 price/rule 明確有 C 型鋼孔加工 / 切工單價時，才另列加工費。
系統訂單中，C 型鋼預設只列材料列；孔洞寫入備註。
workbook 中 C 型鋼的素材長度、素材支數、可裁成品數、餘料長度 / 重量填「不適用-C型鋼鋼捲抽料」。

C 型鋼 kg 計價細則：

若 search_price_candidates 回傳的 C 型鋼 database price row 為 kg 計價，且含單位重：
materialPricingUnit = Kg
materialUnitPrice = database kg 單價
billableQuantity = 單位重 kg/M × 長度M × 支數
subtotal = materialUnitPrice × billableQuantity

使用者說「一支多少」時，「一支」代表支數 = 1；若 price row 是 kg 計價，仍要先換算 kg 數再算小計。
採用價格品名必須保留 database productName；長度不可併入 productName。

範例：

使用者問「C100x50x20x2.3t 6M 一支多少」
database productName = 錏輕型鋼 100*2.3
unitWeight = 4 kg/M
價格B = 26.8 元/kg

正確 workbook 值：
品名 = 錏輕型鋼 100*2.3
數量 = 24
單位 = Kg
單價 = 26.8
小計 = 643.2
備註 = 4 kg/M × 6M × 1支 = 24 kg

不可：
品名 = 錏輕型鋼 100*2.3 6M
單位 = 支
單價 = 624 或 643.2

H 型鋼規則

當 catalog family 或 quote rule 判斷為 H 型鋼 / h_beam：

必須確認尺寸、長度、支數、材質與是否加工。
每支分別算重量與價格。
若 price row 為 kg 價，使用單位重量 × 長度 × 數量 × kg 單價。
常規 / 非常規米數規則必須來自 lookup_quote_rules，不得寫死。
若 quote rule 回傳非常規米數加價，必須列入材料單價或價格調整說明。
H 型鋼切工若有加工需求，優先使用 price candidate 或 cutting rule 的 H 型鋼切工資料。
斜切、翼板切斜、特殊角度、修頭尾或圖面不清時，標低信心並列人工複核。

長條料配料規則

適用長條料，例如：

鋼軌
角鐵
扁鐵
槽鐵
工字鐵
圓鐵
方鋼
圓管
方管
扁方管
其他需裁切的條狀素材

規則：

若客戶指定裁切長度，除非使用者明確說「可切清」、「切清賣」、「只算成品長度」，或 quote rule 明確允許切清，否則一律視為不賣切清。
不賣切清時，材料費不可直接用成品淨長計算。
必須依可售素材長度配料後計價。
必須列素材長度、素材支數、可裁成品數、餘料長度與餘料重量。
若素材長度不明，使用 quote rule 回傳的預設素材長度；若仍不明，可暫估並標低信心。
若 price row 為支價，依素材支數或規則指定支數計價。
若 price row 為 kg 價，依配料後素材總重量或規則指定重量計價，不可直接用成品淨重取代。
若 price row 為 M 價，依規則判斷用素材總長或成品總長；不明時標低信心。
配料結果與報價依據必須寫入報價明細、判讀備註與人工複核。

長條料密度與單位重量：

1. 密度係數使用小數後 3 位：鋼軌 7.859、方鋼 7.852、圓鋼 / 圓鐵 / 圓條 7.851（g/cm3）。
2. 若已有 reviewed 單位重量 kg/m，直接用 kg/m × 長度 m × 數量；不要用密度覆蓋 kg/m。
3. 只有斷面積時，kg/m = 斷面積 cm2 × 密度 ÷ 10。
4. 方鋼只有邊長時，斷面積 cm2 = 邊長 mm × 邊長 mm ÷ 100。
5. 圓鋼只有直徑時，斷面積 cm2 = π × 直徑 mm × 直徑 mm ÷ 400。
6. 鋼軌若缺 kg/m 但有手冊斷面積，使用 7.859 換算；若連斷面積都缺，標低信心，不可用外包矩形面積估鋼軌。
7. 只有 price row 明確為 kg 單價或使用者明確提供售價/kg 時，才用重量 kg × 售價/kg 計算材料費。

板材重量與加工規則

板材重量：

柱底板、連接板、加勁板、補強板、端板、底板、蓋板等圖面材料表板件，都是鐵板加工後的成品名稱，不是不同鋼材品類。
只要圖面或表格可取得長、寬、厚度與數量，就應歸入板材 / 黑板候選並計算材料重量。
板材查價 productName 只能使用 reviewed 產品名；目前可用板材產品名為：鐵板、槽型鐵板、ST 鐵板、2B 鐵板、ST 2B、ST HL、黑鐵板、黑板、錏鐵板、錏板、白鐵板、白鐵。
未標明材質或表面處理時，鋼材預設採用「黑板」或「黑鐵板」reviewed 產品價格候選。不得只因材質未明就讓材料單價或小計停在「未確認」。
SS400 只能作為材質/規格備註，不可當 search_price_candidates.productNames；黑皮板不是產品名，不可作為查價品名或候選。
若後續使用者確認為白鐵、錏板、2B、HL、花紋板、烤漆板或其他材質/表面，再依新材質重新查價修正。
鐵板 / 板材若需算重量，依材料重量規則或 app context 計算。
鐵板 / 黑鐵板 / 黑板的四方理論重密度係數為 7.850（g/cm3）。
四方理論重 kg = 最長邊 mm × 最寬邊 mm × 厚度 mm × 密度 ÷ 1,000,000；黑板未另標材質時用 7.850。
密度、材質與重量計算依據必須來自 reviewed 規則或 app context；若材質未標明但板件明確屬於柱底板、連接板或加勁板等一般鐵板成品，預設使用黑板 / 黑鐵板候選與 7.850 密度係數並標示為暫估，不可改成無價格。
異形、切角、沖孔、折板，若沒有展開尺寸，先用外包四方尺寸並標明。
有展開尺寸時優先用展開尺寸。
材料費計算順序：
1. 先用 search_price_candidates 查「黑板」或「黑鐵板」板材價格候選，並保留採用價格列與單價欄位；必要時再用鐵板、槽型鐵板、ST 鐵板、2B 鐵板、ST 2B、ST HL、錏鐵板、錏板、白鐵板、白鐵等 reviewed 產品名補查。不可用 SS400 或黑皮板作為 productName。
2. 若採用價格列為 kg 單價，用四方理論重 kg × 數量 × kg 單價計算材料費。
3. 若採用價格列為片、才、式或其他單位，依價格列單位與 lookup_quote_rules 回傳規則計算，不可自行轉成其他單位。
4. 材質預設造成的差異寫入判讀備註或低信心原因，但材料費與 subtotal 仍應用最可能的黑板 / 黑鐵板價格暫估。

加工：

切工、孔加工、開槽、折工、其他加工必須分開計算。
加工單價需來自 search_price_candidates 或 lookup_quote_rules。
若加工價格不明，不可填 0，除非 true zero rule 明確成立。
加工與材料在系統訂單中可分列，除非規則要求合併。
