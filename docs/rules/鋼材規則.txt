Steel global rules context

Steel global rules 是 backend 在 provider 前一次載入的 reviewed 鋼材品類、訂單格式、計價方式、配料、true zero、加工與客戶 / 品類規則。

AI 不需要呼叫規則查詢工具取得這些內容；需要品類、加工、配料或計價規則時，直接使用 runtime context 內已提供的 Steel global rules。

每個 reviewed rule 可代表特定客戶、特定品類或特定品名 / 品類推論後的規則，內容至少要讓 AI 判斷：

材料或品類適用範圍。
報價計價單位與計算方式。
配料、素材長度、餘料、是否可切清。
切工、孔加工、開槽、折工與其他加工是否計費。
true zero rule 是否成立。
需要人工確認的缺口與信心降低原因。

公式編號與系統訂單欄位不由單一工具回傳決定；若需要輸出系統訂單資料，AI 依 runtime context、採用價格證據與最終 Markdown 表格整理可核對欄位與計算依據。

backend 不另跑公式計算邏輯，也不以獨立 formula lookup tool 作為 runtime 必要步驟。

口語品名與代號家族補充

3*3鍍鋅方管：可判斷為錏方管 / 方管方向，尺寸候選為 75x75 / 3寸，厚度未指定時需標低信心或列人工確認；若需要代號家族補充資料，方管家族為 GDH。不要因「鍍鋅 / 錏」判成錏板 BNG 或板材切割 DNB。
3分圓鐵：可判斷為圓鐵方向，尺寸候選為 9m/m、3/8、3.3；若需要代號家族補充資料，圓鐵家族為 EQB。

C 型鋼專用規則

當 catalog family 或 quote rule 判斷為：

C 型鋼
鍍鋅 C 型鋼
錏 C 型鋼
亞 C 型鋼
c_type

必須套用 C 型鋼專用規則。

規則：

C / C 型鋼 / C 鋼 / 輕型鋼可先判斷為 C 型鋼 / c_type。
C / C 型鋼 / C 鋼 / 輕型鋼查價時，C、C型鋼、C鋼、輕型鋼是口語 / 品類 evidence；backend 會依 reviewed product-name alias table 多對一補充錏輕型鋼等正式產品品名方向。AI 仍應同時放入本規則推得的產品品名方向與規格片段，例如錏輕型鋼、75*2.3。
C / C 型鋼 / C 鋼 / 輕型鋼未指定材質 / 表面時，預設用錏輕型鋼方向暫報；黑鐵輕型鋼、白鐵輕型鋼只在客戶文字或候選資料明確指向時列為替代或人工確認。
C 型鋼屬於鋼捲抽料 / 成型下料，不視為一般 6M 長條素材裁切料。
不做 6M 長條料配料。
不算素材支數。
不算餘料長度。
不算餘料重量。
材料數量依客戶指定成品長度與支數計算。
若 price row 為 kg 價：用成品總長 × 單位重量 × kg 單價。
若 price row 為 M 價：用成品總長 × M 單價。
若 price row 為支價、式價或其他單位：依價格列單位計算，不可自行轉換。
2C 表示兩支組合，重量與數量需乘 2。
C 型鋼下料長度不另算一般切工。
C 型鋼孔洞預設不另算孔加工費；孔數可記錄在備註。
只有使用者明確說「孔另計」、「切工另計」、「修頭尾」、「特殊切角」，或 price/rule 明確有 C 型鋼孔加工 / 切工單價時，才另列加工費。
系統訂單中，C 型鋼預設只列材料列；孔洞寫入備註。
報價明細中 C 型鋼的素材長度、素材支數、可裁成品數、餘料長度 / 重量可標示「不適用-C型鋼鋼捲抽料」。

C 型鋼 kg 計價細則：

若採用的 C 型鋼 database price row 為 kg 計價，且含單位重：
materialPricingUnit = Kg
materialUnitPrice = database kg 單價
billableQuantity = 單位重 kg/M × 長度M × 支數
subtotal = materialUnitPrice × billableQuantity

使用者說「一支多少」時，「一支」代表支數 = 1；若 price row 是 kg 計價，仍要先換算 kg 數再算小計。
採用價格品名必須保留 database productName；長度不可併入 productName。

範例：

使用者問「C100x50x20x2.3t 6M 一支多少」
database productName = 錏輕型鋼 100*2.3
unitWeight = 4 kg/M
價格B = 26.8 元/kg

正確報價表格值：
品名 = 錏輕型鋼 100*2.3
數量 = 24
單位 = Kg
單價 = 26.8
小計 = 643.2
備註 = 4 kg/M × 6M × 1支 = 24 kg

不可：
品名 = 錏輕型鋼 100*2.3 6M
單位 = 支
單價 = 624 或 643.2

H 型鋼規則

當 catalog family 或 quote rule 判斷為 H 型鋼 / h_beam：

必須確認尺寸、長度、支數、材質與是否加工。
每支分別算重量與價格。
若 price row 為 kg 價，使用單位重量 × 長度 × 數量 × kg 單價。
常規 / 非常規米數規則必須來自 Steel global rules，不得寫死。
若 quote rule 回傳非常規米數加價，必須列入材料單價或價格調整說明。
H 型鋼切工若有加工需求，優先使用 price candidate 或 cutting rule 的 H 型鋼切工資料。
斜切、翼板切斜、特殊角度、修頭尾或圖面不清時，標低信心並列人工複核。

長條料配料規則

適用長條料，例如：

鋼軌
角鐵
扁鐵
槽鐵
工字鐵
圓鐵
方鋼
圓管
方管
扁方管
其他需裁切的條狀素材

規則：

若客戶指定裁切長度，除非使用者明確說「可切清」、「切清賣」、「只算成品長度」，或 quote rule 明確允許切清，否則一律視為不賣切清。
不賣切清時，材料費不可直接用成品淨長計算。
必須依可售素材長度配料後計價。
必須列素材長度、素材支數、可裁成品數、餘料長度與餘料重量。
若素材長度不明，使用 quote rule 回傳的預設素材長度；若仍不明，可暫估並標低信心。
若 price row 為支價，依素材支數或規則指定支數計價。
若 price row 為 kg 價，依配料後素材總重量或規則指定重量計價，不可直接用成品淨重取代。
若 price row 為 M 價，依規則判斷用素材總長或成品總長；不明時標低信心。
配料結果與報價依據必須寫入報價明細、判讀備註與人工複核。
長條料配料與切工計算由 AI 依使用者數量、成品長度、素材長度與 reviewed price row 單位自行計算；backend 不會替你產生固定配料結果。
AI 必須把計算式寫進 Markdown 報價表格或備註，例如 rawMaterialLength、rawMaterialPieceCount、finishedCountPerRawMaterial、remainderLengthOrWeight、cuttingFee、decisionEvidence 或 interpretationNote。
例：1740mm 料 250 支，素材 6M 時，每支素材可裁 floor(6000/1740)=3 支，需 ceil(250/3)=84 支 6M；若 cutting row / 切工規則適用，切工數可用 250 刀並另列切工費。
例：6000mm 料 50 支，素材 6M 時，需 50 支 6M，不另產生裁切刀數，除非使用者或 reviewed rule 指定修頭尾或其他加工。
例：2820mm 料 500 支，素材 6M 時，每支素材可裁 floor(6000/2820)=2 支，需 ceil(500/2)=250 支 6M，餘料需在 evidence 中說明。

長條料密度與單位重量：

1. 密度係數使用小數後 3 位：鋼軌 7.859、方鋼 7.852、圓鋼 / 圓鐵 / 圓條 7.851（g/cm3）。
2. 若已有 reviewed 單位重量 kg/m，直接用 kg/m × 長度 m × 數量；不要用密度覆蓋 kg/m。
3. 只有斷面積時，kg/m = 斷面積 cm2 × 密度 ÷ 10。
4. 方鋼只有邊長時，斷面積 cm2 = 邊長 mm × 邊長 mm ÷ 100。
5. 圓鋼只有直徑時，斷面積 cm2 = π × 直徑 mm × 直徑 mm ÷ 400。
6. 鋼軌若缺 kg/m 但有手冊斷面積，使用 7.859 換算；若連斷面積都缺，標低信心，不可用外包矩形面積估鋼軌。
7. 只有 price row 明確為 kg 單價或使用者明確提供售價/kg 時，才用重量 kg × 售價/kg 計算材料費。

板材重量與加工規則

板材重量：

柱底板、連接板、加勁板、補強板、端板、底板、蓋板等圖面材料表板件，都是鐵板加工後的成品名稱，不是不同鋼材品類。
只要圖面或表格可取得長、寬、厚度與數量，就應歸入板材 / OT 黑鐵候選並計算材料重量。
板材查價只能使用 reviewed 產品名；目前可用板材產品名為：鐵板、槽型鐵板、ST 鐵板、2B 鐵板、ST 2B、ST HL、黑鐵板、黑板、錏鐵板、錏板、白鐵板、白鐵、OT板雷射切割、黑鐵板 雷射切割。
未標明材質或表面處理時，鋼材預設採用黑鐵 OT；查價必須優先使用 OT 雷射切割板材產品候選，不得只因材質未明就讓材料單價或小計停在「未確認」。
SS400 只能作為材質/規格備註，不可當查價產品名；黑皮板不是產品名，不可作為查價品名或候選。
若後續使用者確認為白鐵、錏板、2B、HL、花紋板、烤漆板或其他材質/表面，再依新材質重新查價修正。
鋼材中文名稱對應產品名稱標記：白鐵 = ST；黑鐵 = OT；白鐵沙面 = HL；白鐵霧面 = 2B；白鐵亮面 = BA；白鐵厚度 3mm 以上含 3mm 可使用 NO1 / STNO1 類產品名稱。
NO1 厚度判斷不可只看 `t`，也要接受 `m/m`、`mm` 與產品名稱規格中的板厚，例如 `3.0m/mSTNO1雷射切割`、`4.5m/mSTNO1雷射切割`、`8.0m/mSTNO1切清`、`9.0m/mSTNO1切清`、`STNO1 3.0*4'*8'(73.5)`、`STNO1 3.0*4'*10'(91.8)`、`STNO1 3.0*5'*10'(113)`。
白鐵若沒有明確厚度，先用 ST 查價候選，不可自動套 NO1。
PL 口語板材規格中，`PL` 後面的數字是板厚 mm，`*` 後面的數字是寬度 mm。例如 `PL6*80` 表示 6.0mm 厚、80mm 寬，查價產品名候選為 `6.0m/mOT板雷射切割`；`PL16*96` 表示 16.0mm 厚、96mm 寬，查價產品名候選為 `16.0m/mOT板雷射切割`。
鐵板 / 板材若需算重量，依材料重量規則或 app context 計算。
鐵板 / 黑鐵板 / 黑板的四方理論重密度係數為 7.850（g/cm3）。
四方理論重 kg = 最長邊 mm × 最寬邊 mm × 厚度 mm × 密度 ÷ 1,000,000；黑板 / 黑鐵 / OT 未另標材質時用 7.850。
密度、材質與重量計算依據必須來自 reviewed 規則或 app context；若材質未標明但板件明確屬於柱底板、連接板或加勁板等一般鐵板成品，預設使用 OT / 黑鐵板候選與 7.850 密度係數並標示為暫估，不可改成無價格。
異形、切角、沖孔、折板，若沒有展開尺寸，先用外包四方尺寸並標明。
有展開尺寸時優先用展開尺寸。
材料費計算順序：
1. 優先採用 OT 雷射切割板材價格候選，並保留採用價格列與單價欄位；建議查價 `candidateQueries` 包含厚度型 `6.0m/mOT板雷射切割`、通用 `OT板雷射切割`、`黑鐵板 雷射切割`。不可用 SS400、黑皮板或四方切作為查價關鍵字。
2. 鐵板 / 板材產品價格統一只採用「雷射切割」單價；不得採用「四方切」價格列計算。
3. 鐵板 / 板材計價一律用四方理論重 kg × 數量 × kg 單價計算材料費，即使價格表另有 piece / 片價也不得用片價計算。
4. 若查到的板材雷射切割價格列單位不是 kg，該列不得作為鐵板材料費計價依據；應改找 kg 單價的雷射切割板材價格列，或標為待確認。
5. 材質預設造成的差異寫入判讀備註或低信心原因，但材料費與 subtotal 仍應用最可能的 OT / 黑鐵雷射切割 kg 單價暫估。

加工：

切工、孔加工、開槽、折工、其他加工必須分開計算。
加工單價需來自 reviewed price row 或 reviewed rule。
若加工價格不明，不可填 0，除非 true zero rule 明確成立。
加工與材料在系統訂單中可分列，除非規則要求合併。
